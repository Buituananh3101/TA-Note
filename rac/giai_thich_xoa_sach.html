Cơ Chế Xóa Sách Và Hình Ảnh

Cũng giống như chức năng sửa, Xóa dữ liệu dựa hoàn toàn vào khả năng của Spring Data REST.

📄 File: src/.../dao/SachRepository.java
@RepositoryRestResource(path="sach")
public interface SachRepository extends JpaRepository<Sach, Integer> {
}

API Ngầm Định Của Xóa Sách: Do bạn kế thừa `JpaRepository` và có cài cục `@RepositoryRestResource=sach`, Spring Boot tự động tạo ra một Endpoint xóa dữ liệu tên là DELETE /sach/{id}. Khi Frontend ấn nút Xóa trên giao diện React, React sẽ gửi HTTP DELETE tới URL đó cùng số ID của quyền sách. Backend nhận được sẽ gọi lệnh Xóa mềm hoặc cứng trực tiếp vào Database thông qua Hibernate.

📄 File: src/.../security/Endpoints.java
public static final String[] ADMIN_DELETE_ENDPOINTS = { //123SuaSach
    "/sach/**",    //123SuaSach - Admin được phép DELETE sách
    "/hinh-anh/**", //123SuaSach - Admin được phép DELETE ảnh
};

Cấu Hình Link Xóa: Kẻ thù xâm nhập rất dễ dùng Postman để gọi lệnh Delete, do đó class tĩnh này gom một loạt các Endpoints liên quan tới hành vi xóa thành 1 hạm đội chung có tên là ADMIN_DELETE_ENDPOINTS.

📄 File: src/.../security/SecurityConfiguration.java
http.authorizeHttpRequests(config -> config
     // ...
     .requestMatchers(HttpMethod.DELETE, Endpoints.ADMIN_DELETE_ENDPOINTS)
           .hasAnyAuthority("ROLE_ADMIN")
);

Thiết lập hàng rào bảo vệ: Hàm này kết hợp cùng bộ lọc Jwt Filter. Nó lắng nghe và xét duyệt các Yêu cầu HTTP dạng DELETE. Nó cầm danh sách các URL cần được canh gác, và yêu cầu phải có quyền (Authority) là "ROLE_ADMIN" móc ra từ trong Token giải mã thì cái lệnh `DELETE /sach/id` đó mới được chạy, nếu không sẽ bị ném lỗi `403 Forbidden` (Cấm Truy Cập) hoặc `401 Unauthorized`.
