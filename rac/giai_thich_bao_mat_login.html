Cơ Chế Bảo Mật Đăng Nhập & Đăng Ký (JWT)

📄 Mã hóa mật khẩu khi lưu xuống (Đăng Ký)
// TaiKhoanService.java
String encryptPassword = passwordEncoder.encode(nguoiDung.getMatKhau());
nguoiDung.setMatKhau(encryptPassword);
nguoiDungRepository.save(nguoiDung);

Phòng thủ bị lộ DB: Mọi khách hàng khi Đăng Ký, mật khẩu dạng Plain-text (chữ thường) gõ trên web đều bị hàm BCryptPasswordEncoder băm nát thành hàng chục ký tự mã hóa trước khi ghi vào Database. Đây là hàm Hash một chiều (One-Way Hash), ngay cả Admin cũng không biết pass thật là gì.

📄 Xác thực thông tin lúc Đăng Nhập & Cấp Token
// TaiKhoanController.java
Authentication authentication = authenticationManager.authenticate(
    new UsernamePasswordAuthenticationToken(loginReq.getUsername(), loginReq.getPassword())
);
if(authentication.isAuthenticated()){
    final String jwt = jwtService.generateToken(loginReq.getUsername());
    return ResponseEntity.ok(new JwtResponse(jwt));
}

Quy trình Xác Thực (Authentication): Khi React bấm nút Đăng nhập lên, hàm authenticate của thư viện Spring Security sẽ lấy Mật khẩu gõ vào lúc đó đem đi kiểm tra xem có khớp Hash với trong Database không. Hơn thế, nó kiểm tra các tiêu chí nội bộ: Tài khoản có đang bị vô hiệu lúc gửi mail hay không?

Cấp Thẻ Qua Cửa (JWT): Nếu khớp toàn bộ, hệ thống gọi service tự chế của bạn (JwtService) để đóng dấu mộc lên một chuỗi String lằng nhằng gọi là (JWT Token) rồi trả Token đó xuống lại cho máy React Cất Giữ đi vòng quanh.

📄 Trạm thu phí chằng đường (Bộ Lọc Filter)
// SecurityConfiguration.java
http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);

Xác thực mọi lúc đi qua ranh giới: Sau khi có Token được cấp ở trên. Những lần sau nếu User click vào chức năng nào của web (sửa, xem, xóa giỏ hàng), React sẽ nhét cái cục Token đó vào Headers HTTP gọi là Authorization: Bearer [Token]. Lệnh Cấu hình này xếp cái bộ máy chặn đường có tên là JwtFilter đứng trước các cổng để soi thẻ.

Nếu JWT Filter soi thẻ thấy Thẻ hợp lệ, Không bị mạo danh, Chưa Hết Hạn => Lục ra xem Quyền truy cập là ai, báo cho Spring Security mở cửa nhường đường. Cực kì linh hoạt và gọi là kiến trúc Không Cần Nhớ Lịch Sử (STATELESS).
